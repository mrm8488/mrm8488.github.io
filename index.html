Search Engine

Yes, it seems crazy, but I was ordered to make a search engine (yes, like Google or Bing). I designed an architecture based on microservices. We had some robots (web crawlers made with Node.js) that scraped the Web dinamically. They put in a queue (RabbitMQ) the scraped content to be indexed by another service in a full-text search engine (We used Apache SOLR). The scraped web was marked as scraped with a timestamp (to know when to scrape again) and new links(pages) from that page were pushed to a queue to be scraped (first, it was checked in Redis if they had been scraped previoulsy and how long ago). It was a huge project and I learnt a lot. I designed the whole architecture. I worked as team leader and, of course, I code the most of the project. We did not get something like Google because we had not hardware enough, but the architecture was ready to scale.

  • Client: Megadon
  • Date: May 2016
  • Service: Architect, Team lead and B-E developer
